# JSON vs XML: практичне порівняння

### Вступ
Сьогодні у світі розробки програмного забезпечення існує постійна потреба передавати дані між різними системами, серверами та клієнтами. Для цього використовуються формати обміну даними, серед яких найвідомішими є JSON (JavaScript Object Notation) та XML (eXtensible Markup Language). Хоча обидва формати слугують одній меті — структуруванню інформації для машинного та людського читання, їхній підхід, синтаксис та екосистеми суттєво відрізняються. 

### Практичний приклад серіалізації
Для порівняння форматів візьмемо абстрактний клас `Employee` (Співробітник), написаний мовою C#. Він має 5 властивостей різних типів: число, рядок, десятковий дріб та логічний тип.

```csharp
public class Employee
{
    public int Id { get; set; }
    public string FullName { get; set; }
    public string Department { get; set; }
    public decimal Salary { get; set; }
    public bool IsActive { get; set; }
}
```

Якщо ми створимо об'єкт цього класу та серіалізуємо його, результати виглядатимуть так:

**Результат у форматі JSON:**
```json
{
  "Id": 127,
  "FullName": "Аліна Мороз",
  "Department": "IT",
  "Salary": 75000.00,
  "IsActive": true
}
```

**Результат у форматі XML:**
```xml
<?xml version="1.0" encoding="utf-8"?>
<Employee>
  <Id>127</Id>
  <FullName>Аліна Мороз</FullName>
  <Department>IT</Department>
  <Salary>75000.00</Salary>
  <IsActive>true</IsActive>
</Employee>
```

### Порівняння результатів
Аналізуючи отримані дані, можна одразу помітити головні відмінності. **JSON** є значно компактнішим: він використовує фігурні дужки для об'єктів та пари "ключ-значення". Крім того, JSON зберігає розуміння типів даних (числа та булеві значення пишуться без лапок). **XML**, навпаки, є більш "багатослівним" (verbose). Кожне значення має бути обгорнуте у відкриваючий та закриваючий теги. Це робить файл фізично важчим. Усі дані в XML за замовчуванням сприймаються як текст, поки парсер не перетворить їх згідно зі схемою.

### Сценарії доцільного використання

**Сценарії, де доцільніший JSON:**
1. **RESTful API та Web-додатки:** Оскільки JSON є "рідним" для JavaScript, будь-який сучасний фронтенд (React, Vue, Angular) миттєво перетворює JSON на готові об'єкти без складного парсингу.
2. **NoSQL бази даних:** Такі рішення, як MongoDB, використовують BSON (бінарний JSON) для зберігання даних, оскільки це дозволяє гнучко працювати з документами без жорсткої схеми таблиць.

**Сценарії, де доцільніший XML:**
1. **Enterprise-інтеграції та SOAP-сервіси:** У банківських чи державних системах, де працюють застарілі, але надійні протоколи (SOAP), XML використовується для забезпечення складного шифрування повідомлень та перевірки контрактів.
2. **Документообіг та конфігурації складних систем:** Формати `.docx` (Microsoft Word) або `.svg` (векторна графіка) під капотом є саме XML-файлами. XML дозволяє додавати метадані через "атрибути" тегів, що ідеально підходить для розмітки документів (наприклад, `<Text color="red">Hello</Text>`).

### Чому JSON домінує у Web, а XML залишається в Enterprise?
Успіх **JSON** у веб-розробці пояснюється його легкістю та швидкістю. Веб-додатки часто передають дані через мобільні мережі, де кожен кілобайт має значення. JSON менший за обсягом, швидше серіалізується/десеріалізується і не потребує зовнішніх бібліотек у браузері. Це зробило його абсолютним стандартом для створення сучасних, швидких клієнт-серверних архітектур.

З іншого боку, **XML** зберігає свої позиції в корпоративному (enterprise) середовищі завдяки своїй строгості. XML має потужний інструмент — **XSD (XML Schema Definition)**. Це дозволяє компаніям створювати жорсткі правила (контракти) щодо того, які дані можуть бути передані, якого вони мають бути типу та формату. Якщо банк передає транзакцію іншому банку, система автоматично "відкине" XML-документ ще до етапу обробки, якщо він не відповідає узгодженій схемі. Крім того, XML підтримує простори імен (namespaces), що дозволяє уникати конфліктів даних при злитті інформації з різних систем. 

### Висновок
Підсумовуючи, JSON та XML є інструментами для різних завдань. JSON — це швидкість, легкість та ідеальна сумісність із веб-технологіями. XML — це строгість, безпека та надійні контракти для складних корпоративних систем.
